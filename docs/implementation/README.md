# Implementation Notes

This folder is for execution records, not high-level planning.

Use one markdown file per revision so the project history stays inspectable.

## Naming convention

Preferred pattern:

- `phase-0/rev-01-project-bootstrap.md`
- `phase-0/rev-02-config-and-camera-abstraction.md`
- `phase-1/rev-01-detection-baseline.md`

## When to create a new revision note

Create a new note when:

- scope changes materially
- you start a new implementation slice
- testing reveals a change in plan
- a revision has a clear new goal or exit criterion

## What belongs here

- current revision goal
- tasks for the revision
- testing plan
- observed issues
- decisions made during implementation
- result summary after the revision is done

Roadmap and architecture material should stay outside this folder.
