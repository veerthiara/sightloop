# Developer Notes

This directory separates long-lived planning from execution notes.

- `roadmap/` contains phase and project planning documents
- `architecture/` contains cross-cutting design and workflow guidance
- `implementation/` contains revision-by-revision build notes as the project progresses

Use roadmap docs to decide what to build.

Use implementation docs to record how each revision was actually executed and what was learned.

## Site purpose

This site is the working documentation hub for SightLoop.

It is intentionally developer-facing rather than end-user-facing.

## How to navigate

- Start in `Roadmap` for project direction and phase sequencing.
- Use `Architecture` for structure, technical decisions, and working conventions.
- Use `Implementation` for revision-by-revision execution notes.
